# CRUD API (Users & Listings)

Small Spring Boot + PostgreSQL CRUD service with Maven Wrapper.

## Run
- Prereqs: Java 17, Postgres running, DB settings in `src/main/resources/application.properties`.
- Start: `cd crud` then `.\mvnw.cmd spring-boot:run` (PowerShell) or `mvnw.cmd spring-boot:run` (cmd).
- Base URL: `http://localhost:8080`

## Users endpoints (`/api/users`)
- **Create**: `POST /api/users`
  ```json
  { "username": "alice", "email": "alice@example.com", "password": "secret" }
  ```
  Response: 201 Created (returns user or message).
- **Get by id**: `GET /api/users/{id}`
- **List all**: `GET /api/users`
- **Update**: `PUT /api/users/{id}`
  ```json
  { "username": "alice2", "email": "alice2@example.com", "password": "secret" }
  ```
  Response: 200 OK with updated user.
- **Delete**: `DELETE /api/users/{id}`

## Listings endpoints (`/api/listings`)
- **Create**: `POST /api/listings`
  ```json
  { "title": "Bike", "description": "Road bike", "price": 250.0 }
  ```
- **Get by id**: `GET /api/listings/{id}`
- **List all**: `GET /api/listings`
- **Update**: `PUT /api/listings/{id}`
  ```json
  { "title": "Bike", "description": "Upgraded", "price": 275.0 }
  ```
- **Delete**: `DELETE /api/listings/{id}`

## Notes
- IDs are generated by the database; omit `id` in POST bodies.
- If you see “null value in column id”, ensure the `users` and `listings` tables use identity/serial IDs (or drop/recreate with the app’s JPA mappings).
